Web сервис "Списки студентов"
1) Получить всех студентов. Есть версии как для Get, так и для POST запросов.
Параметры:
	date - дата, начиная с которой нужно получить изменившихся студентов (например, изменился статус из-за отчисления).
	Если date не указывается, то выгружаются все студенты.
	Пример: ?date=20220102.
Возвращает Json массив:
[
{
	hexKey - уникальный id студента;
	firstN - имя;
	lastN - фамилия;
	patrnmc - отчество;
	status - статус (обучается, отчислен, академ и т.д);
	group - номер группы;
	tel - телефон;
	email - электронная почта;
	dep - кафедра;
	lang[] - изучаемые языки
	langGroup[] - номера групп по этим языкам
}
]

2) Получить студентов для системы Фото БГЭУ.
В БГЭУ существует прога для фото + БД к ней (id + foto). id и прочую инфу она берёт из view mySQL базы.
Вот именно для этих вьюшек мы накидали этот сервис. JSON парситься на стороне сервера с mySQL. Частота обращей к сервису +- раз в день.
Сервис пока не запустили, т.к. пока нет разрешения проректора.
Возвращает Json массив:
[
{
	rowguid - id студента (Число);
	IsActive - активен ли сутдент (Число. 1 - обучается, 0 - академ и прочее);
	FIO - ФИО одной строкой (Строка 150);
	Name_F - Фамилия (Строка 50);
	Name_I - Имя (Строка 50);
	Name_O - Отчество (Строка 50);
	STDNUM - Номер зачетной книжки (Строка 10);
	PasportS - Серия паспорта (Строка 10);
	PasportN - Номер паспорта (Строка 10);
	Passport - Серия + номер паспорта (Строка 20);
	PERSID - Личный номер паспорта (Строка 20);
	idF - id факультета (Число);
	FACULTY - факультет (Строка 50);
	IdFormaTime - id формы обучения (Число);
	STUDYFORM = форма обучения (Строка 50);
	COURSE - курс (Число);
	YEAR_ - год набора (Дата);
	Pol - пол (Число 0 - муж, 1 - жен);	
	UchYear - текущий учебный год (Строка 4);
	ISSUEDDATE - год выпуска (Дата);	
	DateZach - дата зачисления (Дата);
	GROUP_ - группа (Строка 50);
	SPECIALITY - специальность (Строка 50);
	IDSPECIALITY - id специальности (Число);
}
]
